PREEMPT_RT kernel rollout — AGX Thor fleet runbook
Status: ch4 = validated canary (staged on 2026-06-26, pending supervised reboot). Apply this to each remaining Thor box one at a time. Never batch-reboot a control fleet.

Target end state per box:

Kernel 6.8.12-rt-tegra-38.4.0 (uname -v shows SMP PREEMPT_RT), booted by default.
L4T BSP at r38.4 (kernel must match userspace release — do not mix r38.2 userspace with r38.4 kernel).
CPU isolation args preserved on both extlinux entries.
MAXN power mode + jetson_clocks at every boot.
Kernel packages held so apt upgrade can't silently replace the RT kernel.
0. Why this runbook branches (read once)
The reference doc assumes the box is already on r38.4. Real fleet boxes vary:

L4T release — a box on r38.2 needs a BSP upgrade first; a box already on r38.4 skips straight to the RT kernel. Check before doing anything.
A dist-upgrade is NOT what you want. On ch4 it wanted 422 packages — only 56 were the NVIDIA BSP, the other 366 were unrelated accumulated Ubuntu noble updates. Upgrade only nvidia-l4t* (0 forced Ubuntu deps). Leave OS updates to a separate maintenance window.
extlinux.conf is regenerated by /etc/kernel/postinst.d/xx-nvidia-update-extlinux (it is not a dpkg conffile, so --force-confold does NOT protect it). On ch4 the postinst happened to preserve the primary APPEND and copy it into the new real-time entry — but verify and re-apply on every box.
nvpmodel -m 0 fails on the old running kernel (/sys/class/devfreq/bwmgr/max_freq doesn't exist until you're on the new kernel). Set the boot default instead so nvpmodel.service applies MAXN after reboot.
Custom CRC-locked OOT modules (custom iwlwifi §3, Stereolabs §4) must be rebuilt against /lib/modules/6.8.12-rt-tegra or they won't load. ch4 had neither — other boxes may. Check per box.
The reboot applies a UEFI firmware capsule (/boot/efi/EFI/UpdateCapsule/*.cap, staged by the bootloader pkg upgrade). It MUST complete on stable power — this is the one brick-capable step. Supervise it.
1. Pre-flight (read-only — run on each box, decide the path)
# Must be a Thor:
cat /proc/device-tree/model        # expect: NVIDIA Jetson AGX Thor ...
# Current L4T release of the userspace/BSP:
grep -m1 'r38' /etc/apt/sources.list.d/nvidia-l4t-apt-source.list
dpkg-query -W -f='${Version}\n' nvidia-l4t-core      # 38.2.x => need §2 ; 38.4.x => skip to §3
uname -v                                              # PREEMPT_RT already? then box may be done
# CRC-locked custom modules that will need rebuilding (§5) if present:
find /lib/modules/$(uname -r) -iname 'iwlwifi*'; dkms status 2>/dev/null
find /lib/modules/$(uname -r) -ipath '*stereolabs*' -o -ipath '*zed*' 2>/dev/null
# Record the existing isolation args (re-applied in §4):
grep -m1 APPEND /boot/extlinux/extlinux.conf \
  | grep -oE '(nohz_full=[^ ]+|rcu_nocbs=[^ ]+|rcu_nocb_poll|skew_tick=[^ ]+|irqaffinity=[^ ]+|isolcpus=[^ ]+)'
Decision:

nvidia-l4t-core is 38.2.x → do §2 then §3.
nvidia-l4t-core is already 38.4.x → skip §2, go to §3.
Back up first:

BK=/root/rt-kernel-upgrade-backup-$(date +%F)
sudo mkdir -p "$BK"
sudo cp -a /boot/extlinux/extlinux.conf "$BK"/extlinux.conf.bak
sudo cp -a /etc/apt/sources.list.d/nvidia-l4t-apt-source.list "$BK"/
sudo cp -a /etc/nvpmodel/$(readlink /etc/nvpmodel.conf | xargs basename) "$BK"/ 2>/dev/null
dpkg -l | grep nvidia-l4t | sudo tee "$BK"/dpkg-nvidia-l4t-BEFORE.txt >/dev/null
2. BSP upgrade r38.2 → r38.4 (ONLY if box is on r38.2)
# Point the existing NVIDIA source at r38.4 (3 lines: common, som, ffmpeg):
sudo sed -i 's# r38\.2 main# r38.4 main#' /etc/apt/sources.list.d/nvidia-l4t-apt-source.list
sudo apt-get update

# Sanity: confirm the BSP delta is ~56 nvidia-l4t pkgs, and how much Ubuntu churn you're skipping:
sudo apt-get dist-upgrade -s | grep -E '[0-9]+ upgraded'            # the big number (incl. Ubuntu) — do NOT run this
NVPKGS=$(sudo apt-get dist-upgrade -s | grep -oE '^Inst nvidia-l4t[a-z0-9-]+' | awk '{print $2}')

# BSP-ONLY upgrade (keeps local conffiles; 0 forced Ubuntu deps):
sudo DEBIAN_FRONTEND=noninteractive apt-get install -y --only-upgrade \
  -o Dpkg::Options::=--force-confdef -o Dpkg::Options::=--force-confold $NVPKGS

# Verify all nvidia-l4t now 38.4.0 and apt is consistent:
dpkg -l | grep nvidia-l4t | awk '$3 !~ /38.4.0/ && $3 ~ /^38\./ {print}'   # expect empty
sudo apt-get -f install -s | grep -E '[0-9]+ upgraded'                      # expect 0 upgraded
A firmware capsule is now staged under /boot/efi/EFI/UpdateCapsule/. Expected.

3. Install the PREEMPT_RT kernel (r38.4)
echo "deb https://repo.download.nvidia.com/jetson/rt-kernel r38.4 main" \
  | sudo tee /etc/apt/sources.list.d/nvidia-l4t-rt-kernel.list
sudo apt-get update
sudo DEBIAN_FRONTEND=noninteractive apt-get install -y \
  -o Dpkg::Options::=--force-confdef -o Dpkg::Options::=--force-confold \
  nvidia-l4t-rt-kernel nvidia-l4t-rt-kernel-headers \
  nvidia-l4t-rt-kernel-oot-modules nvidia-l4t-display-rt-kernel nvidia-l4t-rt-kernel-openrm

# Verify:
ls -la /boot/Image.real-time                                  # exists
find /lib/modules/6.8.12-rt-tegra -name '*.ko' | wc -l        # ~1500+
grep -E '^DEFAULT|^LABEL' /boot/extlinux/extlinux.conf        # DEFAULT real-time; primary fallback kept
Hold kernel packages (RT set + stock kernel + dtbs):

sudo apt-mark hold nvidia-l4t-rt-kernel nvidia-l4t-rt-kernel-headers \
  nvidia-l4t-rt-kernel-oot-modules nvidia-l4t-display-rt-kernel nvidia-l4t-rt-kernel-openrm \
  nvidia-l4t-kernel nvidia-l4t-kernel-dtbs
apt-mark showhold        # expect 7
4. Re-apply / verify CPU isolation on BOTH entries
ISO='nohz_full=11 rcu_nocbs=11-13 rcu_nocb_poll skew_tick=1 irqaffinity=0-10 isolcpus=managed_irq,11-13'
# Confirm both 'primary' and 'real-time' APPEND lines contain isolcpus + nohz_full:
awk '/LABEL (primary|real-time)/{l=$2} /^[[:space:]]*APPEND/{printf "%s: %s\n", l, (/isolcpus/&&/nohz_full/?"OK":"MISSING")}' \
  /boot/extlinux/extlinux.conf
If either says MISSING, append $ISO to that entry's APPEND line (edit /boot/extlinux/extlinux.conf).

Note: isolation is an optional tail-latency tweak (§2 of the source doc), not required for the RT kernel to be real-time. Keep parity with the fleet baseline (cores 11–13 isolated on a 14-core T5000).

⚠ Finding from ch4 (applies to the whole fleet): NVIDIA's prebuilt 6.8.12-rt-tegra is NOT built with CONFIG_NO_HZ_FULL or CONFIG_RCU_NOCB_CPU (only NO_HZ_IDLE, HZ=250, PREEMPT_RT=y). So nohz_full=, rcu_nocbs=, and rcu_nocb_poll on the cmdline are silently ignored — no /sys/devices/system/cpu/nohz_full file, no rcuo* kthreads. Only isolcpus=managed_irq, irqaffinity, and skew_tick actually take effect. This is harmless but the cmdline misrepresents the real isolation. Determinism comes from PREEMPT_RT + isolcpus + locked max clocks: on ch4, cyclictest pinned to isolated core 11 under full housekeeping load measured 6 µs max. If true full-tickless on the RT core is required, a custom-built RT kernel is needed (the prebuilt one can't provide it). Quick check per box: (zcat /proc/config.gz||cat /boot/config-$(uname -r)) | grep -E 'NO_HZ_FULL|RCU_NOCB'

5. Rebuild CRC-locked OOT modules — ONLY if pre-flight found them
If the box has custom iwlwifi (§3) or Stereolabs/ZED (§4) modules, they are locked to the exact kernel build and must be rebuilt against the RT modules dir before reboot, else Wi-Fi / cameras break:

# Build each against /lib/modules/6.8.12-rt-tegra (use that box's existing §3/§4 build scripts),
# then: sudo depmod 6.8.12-rt-tegra
ch4 had neither, so this step was a no-op there. Verify per box.

6. Tuning (MAXN + jetson_clocks) — persistent, applies after reboot
nvpmodel -m 0 fails on the old running kernel (missing bwmgr devfreq node), so set the boot default here:

CONF=$(readlink -f /etc/nvpmodel.conf)
sudo sed -i 's/< PM_CONFIG DEFAULT=1 >/< PM_CONFIG DEFAULT=0 >/' "$CONF"   # 1=120W -> 0=MAXN
grep 'PM_CONFIG DEFAULT' "$CONF" | grep -v '^#'
⚠ Not enough on its own (ch4 lesson): nvpmodel honors its saved runtime state (/var/lib/nvpmodel/status, e.g. pmode:0001) over the config DEFAULT, so the box still boots 120 W the first time. You must also run sudo nvpmodel -m 0 after the reboot (see §8) to update the saved state and bring all 14 cores to full power. After that it persists. Persistent jetson_clocks service:

sudo tee /etc/systemd/system/jetson_clocks.service >/dev/null <<'EOF'
[Unit]
Description=Maximize Jetson clocks for real-time determinism
After=nvpmodel.service
Wants=nvpmodel.service
[Service]
Type=oneshot
ExecStart=/usr/bin/jetson_clocks
RemainAfterExit=yes
[Install]
WantedBy=multi-user.target
EOF
sudo systemctl daemon-reload && sudo systemctl enable jetson_clocks.service
7. Reboot (⚠ applies firmware capsule — supervise, stable power)
sudo reboot
Do not cut power. First boot applies the UEFI firmware capsule, then boots the RT kernel.

8. Post-reboot validation
uname -v                                  # must contain: SMP PREEMPT_RT
cat /etc/nv_tegra_release | head -1       # R38 ... REVISION 4.x

# REQUIRED: force MAXN now (config DEFAULT alone does NOT win over saved nvpmodel state).
sudo nvpmodel -m 0
sudo nvpmodel -q | grep -i 'power mode'   # must now read: MAXN
cat /var/lib/nvpmodel/status              # must read: pmode:0000  (persists from here on)
nproc; cat /sys/devices/system/cpu/online # 14 cores online (0-13)
sudo jetson_clocks                        # re-apply across all cores once MAXN is active
systemctl is-active jetson_clocks.service # active (exited)

# Latency under load. Two tests:
sudo apt-get install -y rt-tests stress-ng
# (a) general pool:
stress-ng --cpu $(nproc) --timeout 25s &
sudo cyclictest -p99 -t$(nproc) -m -D20 -q
# (b) the meaningful one — isolated RT core 11 while housekeeping cores 0-10 are hammered
#     (ch4 result: Max 6us):
taskset -c 0-10 stress-ng --cpu 11 --timeout 30s &
sudo cyclictest -p99 -a11 -t1 -m -D20 -q
Drop-in self-test (writes a PASS/FAIL report, runs cyclictest, then disables itself) — copy ch4's /root/rt-postboot-validation.sh + rt-postboot-validation.service, systemctl enable it before the §7 reboot, then read /root/rt-postboot-validation-LATEST.log after boot. Note: that script's MAXN check will FAIL on first boot for the reason above — run nvpmodel -m 0 then re-run the script.

9. Rollback (if RT kernel misbehaves)
At the boot menu (TIMEOUT 30) select primary — the stock kernel is retained as fallback.
To make it permanent: set DEFAULT primary in /boot/extlinux/extlinux.conf (or restore $BK/extlinux.conf.bak).
The BSP r38.4 upgrade and firmware capsule are not rolled back this way; treat the BSP bump as one-way (re-flash to downgrade).
